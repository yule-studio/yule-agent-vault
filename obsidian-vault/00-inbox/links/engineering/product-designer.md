## Reference 링크

- [Refactoring UI (Adam Wathan, Steve Schoger)](https://www.refactoringui.com/) — 실용 디자인 7 원칙 (책 + 영상)
- [Material Design 3](https://m3.material.io/) — Google 디자인 시스템 표준
- [Apple HIG](https://developer.apple.com/design/human-interface-guidelines/) — Apple 인간 인터페이스 가이드라인
- [Nielsen Norman Group](https://www.nngroup.com/articles/) — UX 리서치 권위 출처
- [Laws of UX](https://lawsofux.com/) — UX 법칙 카드 (Fitts / Hick / 등)
- [A11y Project](https://www.a11yproject.com/) — 접근성 체크리스트
- [WCAG 2.2 Quick Reference](https://www.w3.org/WAI/WCAG22/quickref/) — 접근성 표준
- [Smashing Magazine](https://www.smashingmagazine.com/articles/) — UX / UI 패턴 깊은 글
- [Figma Best Practices](https://help.figma.com/hc/en-us/categories/360002051333) — 디자인 협업 표준
- [UX Collective Medium](https://uxdesign.cc/) — 디자인 산업 트렌드
- [Adobe XD 디자인 시스템 가이드](https://xd.adobe.com/ideas/principles/design-systems/) — 디자인 시스템 구축 단계
- [Component-driven UI (Storybook)](https://www.componentdriven.org/) — 컴포넌트 우선 개발 철학
- [Inclusive Design Principles (Microsoft)](https://inclusive.microsoft.design/) — 포용적 디자인
- [Design Better (InVision)](https://www.designbetter.co/) — 디자인 시스템 케이스 스터디
- [Atomic Design (Brad Frost)](https://atomicdesign.bradfrost.com/) — atom/molecule/organism 패턴의 표준

## 운영 규칙

1. 링크 검증 — 죽은 링크는 발견 시 즉시 제거.
2. 새 링크 추가 시 한 줄 설명 (왜 좋은지) 필수.
3. 깊은 학습 자료는 `30-resources/` 에 별도 노트로 추출 + 본 카탈로그에는 링크만.
