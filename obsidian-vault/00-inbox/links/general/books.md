## Reference 링크

- [Designing Data-Intensive Applications (Martin Kleppmann)](https://dataintensive.net/) — 분산 시스템 표준 책
- [The Pragmatic Programmer (20th anniv)](https://pragprog.com/titles/tpp20/the-pragmatic-programmer-20th-anniversary-edition/) — 실용 개발자 표준
- [Software Engineering at Google (book)](https://abseil.io/resources/swe-book) — Google 의 SWE 운영 표준 (무료 공개)
- [Accelerate (Forsgren, Humble, Kim)](https://itrevolution.com/product/accelerate/) — DORA metric 의 책
- [Effective Java (Joshua Bloch)](https://www.oreilly.com/library/view/effective-java-3rd/9780134686097/) — Java 90 항목
- [Designing Distributed Systems (Brendan Burns)](https://azure.microsoft.com/en-us/resources/designing-distributed-systems/) — 분산 패턴 책 (무료)
- [Refactoring 2nd (Martin Fowler)](https://martinfowler.com/books/refactoring.html) — 리팩터링의 표준
- [Building Microservices (Sam Newman)](https://samnewman.io/books/building_microservices_2nd_edition/) — MSA 책
- [Site Reliability Engineering (Google)](https://sre.google/sre-book/table-of-contents/) — SRE 책 (무료 공개)
- [A Philosophy of Software Design (Ousterhout)](https://web.stanford.edu/~ouster/cgi-bin/aphilosophyofsoftwaredesign.php) — 복잡성 관리

## 운영 규칙

1. 링크 검증 — 죽은 링크는 발견 시 즉시 제거.
2. 새 링크 추가 시 한 줄 설명 (왜 좋은지) 필수.
3. 깊은 학습 자료는 `30-resources/` 에 별도 노트로 추출 + 본 카탈로그에는 링크만.
