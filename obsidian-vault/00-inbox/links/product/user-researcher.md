## Reference 링크

- [NN/g — User Research methods](https://www.nngroup.com/articles/which-ux-research-methods/) — 20 가지 리서치 방법 매트릭스
- [Interaction Design Foundation](https://www.interaction-design.org/literature/topics) — UX 리서치 권위 textbook
- [User Interviews — knowledge base](https://www.userinterviews.com/blog) — 인터뷰 운영 실무
- [Just Enough Research (Erika Hall)](https://abookapart.com/products/just-enough-research) — 리서치 책 (요약본 다수)
- [Maze blog — UX research](https://maze.co/blog/) — usability test 운영
- [Hotjar 가이드](https://www.hotjar.com/blog/) — 사용자 행동 관찰 / heat map
- [Jobs To Be Done framework](https://jobs-to-be-done.com/) — JTBD 의 표준 출처
- [Persona templates (NN/g)](https://www.nngroup.com/articles/persona/) — 페르소나 작성 가이드
- [Journey Map 표준 (NN/g)](https://www.nngroup.com/articles/journey-mapping-101/) — journey map 양식
- [Dovetail blog](https://dovetail.com/blog/) — 리서치 인사이트 운영
- [Research repository 운영 (ResearchOps)](https://researchops.community/) — 리서치 팀 운영

## 운영 규칙

1. 링크 검증 — 죽은 링크는 발견 시 즉시 제거.
2. 새 링크 추가 시 한 줄 설명 (왜 좋은지) 필수.
3. 깊은 학습 자료는 `30-resources/` 에 별도 노트로 추출 + 본 카탈로그에는 링크만.
