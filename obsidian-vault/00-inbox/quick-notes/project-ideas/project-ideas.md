## 노트 표준 구조

각 아이디어 노트는 7 섹션:

1. **개요** — 한 문단 설명
2. **수익 모델** — freemium / Pro / 광고 / 후원 등 + 가격대
3. **실제 레퍼런스** — 운영 중인 서비스 + 추정 ARR + 운영자 정보
4. **기술 스택** — 1주-4주 MVP 가능한 도구
5. **진입 장벽** — 낮음/중간/높음 + 차별화 방향
6. **한국 시장 변형** — 본인인증 / 결제 / 한국어 NLP / 채널 통합
7. **학습 자료** — 운영자 인터뷰 + open-source 레포 + 마케팅 playbook

## 운영 규칙

1. 아이디어는 idea 단계. 채택 시 status `adopted` + `_moc/<project>.md` hub 신설.
2. 아이디어 검증 단계 (idea → research → MVP → launch) 별 status 갱신.
3. 한 분기 시도 결과 (성공/실패/pivot) 를 본 노트 footer 에 기록.
4. 비슷한 아이디어가 또 떠오르면 본 노트 `## 변형` 섹션에 추가 (중복 노트 X).
