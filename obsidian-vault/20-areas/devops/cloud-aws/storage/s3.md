## 1. 한 줄

**object storage** — 사실상 무제한 / 99.999999999% (11 nines) durability / URL 로 접근.
AWS 의 시작 (2006) + 인터넷의 backbone.

---

## 2. 왜

- **EBS 등 block** = OS 가 디스크처럼 — 인스턴스 attached
- **EFS** = NFS — POSIX file
- **S3** = HTTP REST — 무제한 / DR / 글로벌

쓰임:
- 정적 웹 / 이미지 / 비디오
- 백업 / archive
- data lake (분석 raw)
- 로그
- container image (ECR 의 backing)
- artifact (CI/CD)

---

## 3. 핵심 개념

| 개념 | 의미 |
| --- | --- |
| **Bucket** | 최상위 컨테이너 (글로벌 unique 이름) |
| **Object** | 파일 (key + body + metadata + version) |
| **Key** | bucket 안의 경로 (`folder/file.png`) |
| **Storage Class** | 비용 / 접근 속도 등급 |
| **Versioning** | 같은 key 의 여러 버전 |
| **Lifecycle** | 자동 cold / 만료 |
| **Replication** | bucket 간 자동 복사 |
| **Pre-signed URL** | 시한 URL (업로드 / 다운로드) |
| **Event Notification** | 이벤트 → Lambda / SQS / SNS |

---

## 4. Storage Class

| Class | 비용 ($/GB·월) | 접근 | 용도 |
| --- | --- | --- | --- |
| Standard | $0.023 | 즉시 | 핫 데이터 |
| Standard-IA | $0.0125 | 즉시 + retrieval 비용 | 가끔 |
| One Zone-IA | $0.01 | 1 AZ 만 | 백업 (다른 곳 복제 있음) |
| Intelligent-Tiering | 동적 | 즉시 | 알 수 없는 패턴 (자동 분류) |
| Glacier Instant | $0.004 | 즉시 + 비싼 retrieval | archive |
| Glacier Flexible | $0.0036 | 1분~12시간 | 백업 |
| Glacier Deep Archive | $0.00099 | 12-48시간 | 규제 / 장기 |

→ 일반 = Standard / Intelligent-Tiering. 백업 = Deep Archive.

## 5. 설치 / 사용

### 5.1 Console
S3 → Create bucket → 이름 + region.

### 5.2 CLI

```bash
# bucket 생성
aws s3 mb s3://my-bucket --region ap-northeast-2

# 업로드
aws s3 cp file.txt s3://my-bucket/folder/
aws s3 sync ./local/ s3://my-bucket/dir/

# 다운로드
aws s3 cp s3://my-bucket/file.txt .
aws s3 sync s3://my-bucket/ ./local/

# 목록
aws s3 ls s3://my-bucket/folder/

# 삭제
aws s3 rm s3://my-bucket/file.txt
aws s3 rb s3://my-bucket --force            # bucket + 내용

# pre-signed URL
aws s3 presign s3://my-bucket/file.txt --expires-in 3600
```

### 5.3 SDK (Python)

```python
import boto3
s3 = boto3.client("s3")

s3.upload_file("file.txt", "my-bucket", "folder/file.txt")
s3.download_file("my-bucket", "folder/file.txt", "file.txt")

# stream
with open("file.txt", "rb") as f:
    s3.put_object(Bucket="my-bucket", Key="folder/file.txt", Body=f)

# pre-signed
url = s3.generate_presigned_url("get_object",
    Params={"Bucket":"my-bucket","Key":"file.txt"}, ExpiresIn=3600)
```

### 5.4 Terraform

```hcl
resource "aws_s3_bucket" "data" {
  bucket = "my-data-bucket"
}

resource "aws_s3_bucket_versioning" "data" {
  bucket = aws_s3_bucket.data.id
  versioning_configuration { status = "Enabled" }
}

resource "aws_s3_bucket_lifecycle_configuration" "data" {
  bucket = aws_s3_bucket.data.id
  rule {
    id     = "archive-old"
    status = "Enabled"
    transition {
      days          = 30
      storage_class = "STANDARD_IA"
    }
    transition {
      days          = 90
      storage_class = "GLACIER"
    }
    expiration {
      days = 365
    }
  }
}

resource "aws_s3_bucket_public_access_block" "data" {
  bucket                  = aws_s3_bucket.data.id
  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```

## 6. 보안 — public 절대 X (의도 아니면)

```bash
# 모든 public 차단
aws s3api put-public-access-block --bucket my-bucket \
  --public-access-block-configuration \
  BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

⚠️ public bucket 의 의도하지 않은 leak = 거의 매주 뉴스. Block Public Access 기본 활성.

### 6.1 Bucket Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "AllowSpecificRole",
    "Effect": "Allow",
    "Principal": { "AWS": "arn:aws:iam::111222333444:role/AppRole" },
    "Action": ["s3:GetObject", "s3:PutObject"],
    "Resource": "arn:aws:s3:::my-bucket/*"
  }]
}
```

### 6.2 암호화

```bash
# SSE-S3 (기본, 2023+ 자동)
# SSE-KMS — KMS key
aws s3api put-bucket-encryption --bucket my-bucket --server-side-encryption-configuration '{
  "Rules":[{
    "ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"aws:kms",
      "KMSMasterKeyID":"arn:aws:kms:..."
    }
  }]
}'
```

## 10. Performance

- **3,500 PUT / 5,500 GET per second per prefix**
- 같은 prefix 폭증 = throttle → key 앞에 random prefix
- multipart upload — 5 MB+ 권장, 5 GB+ 필수
- S3 Transfer Acceleration — CloudFront edge 로 업로드
- **S3 Select** — 일부 (CSV/JSON) 만 fetch — 비용 ↓

---

## 11. 비용 — 4 가지 축

```
1. Storage   ($/GB·월) — 위 class 표
2. Request   ($/1K) — PUT $0.005 / GET $0.0004 등
3. Transfer  ($/GB) — 외부로 나감 ($0.09/GB), 내부 무료, CloudFront 통해 더 저렴
4. Retrieval — Glacier 등 ($)
```

NAT Gateway 비용 회피 → **VPC Endpoint (gateway)** for S3 (무료).

## 12. 함정

### 12.1 Public 노출
의도치 않은 leak. Block Public Access ON.

### 12.2 versioning + delete 비용
delete = marker 만 — 실제 데이터 유지 → storage 비용 계속. Lifecycle expiration 필수.

### 12.3 listing 의 한계
한 번에 1000 keys. paginate.

### 12.4 directory 환상
key prefix 만. 진짜 디렉토리 X. 빈 "디렉토리" 생성은 0-byte object.

### 12.5 last-write-wins
strongly consistent (2020+) — 같은 key 의 동시 PUT 은 마지막.

### 12.6 큰 파일의 single PUT
5 GB 한계. Multipart upload 필수.

### 12.7 NAT Gateway 통과 비용
S3 endpoint (gateway VPC endpoint) 무료.

### 12.8 region 따라 다른 endpoint
잘못된 region 에서 SigV4 실패.
